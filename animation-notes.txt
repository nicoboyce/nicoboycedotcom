CHARACTER 1: KETTLEPHANT
-------------------------
Personality: Naive, dumb, cheerful, puppy-like energy
Movement style: Bouncy, enthusiastic, bounding like a puppy

Animation Pattern: Rocking Horse Motion
- Front legs (FL + FR) move together
- Back legs (BL + BR) move together
- Front and back pairs oppose each other (forward rock vs backward rock)
- Left-right asymmetry added (right legs rotate 2-3째 more than left)
- Creates wobblier, less robotic motion

Body Parts (separate SVG files):
- body.svg (main kettle shape)
- eyes.svg
- mouth.svg (tongue) - stretches on bounce peaks (frames 7, 11)
- handle.svg
- leg.svg (reused for all 4 legs: fl, fr, bl, br)
- note.svg (musical note - pulses with bounces)

Musical Note Behaviour:
- Small/faded when kettlephant squashes down
- Large/bright when stretched up (synced with bounce peaks)
- Peak brightness at frames 7 and 11

12-Frame Bounce Cycle:
- Frames 1-4: Forward rock (front legs back, back legs forward)
- Frame 5: Neutral stretch
- Frames 6-8: Backward rock (front legs forward, back legs back)
- Frame 9: Neutral
- Frames 10-12: Big forward rock

Key Animation Values:
- Front legs swing up to 15-18째 forward at peak (frames 6-8)
- Back legs swing in opposition: -15-18째 at same peaks
- Body squash: scaleY(0.92) at deepest (frame 11)
- Body stretch: scaleY(1.02) at neutral peak (frame 5)
- Mouth: translateY(10px) scale(1.02-1.03) on big bounces (frames 7, 11)

Animation Refinements:
- Added left-right asymmetry: right legs rotate 2-3째 more than left
- Creates wobblier, more organic motion (less robotic)
- Front legs increased forward angle for more pronounced puppy-bound
- Rocking horse motion: front pair + back pair move together, oppose each other

TECHNICAL STRUCTURE
-------------------
File: demo.md
Layout: blank (no site navigation)
Container: 800x800px at left: 64%
Scroll distance: 1500vh for 60 frames (12-frame pattern x5)

Frame Structure:
- 60 total frames (12-frame bounce cycle repeated 5 times)
- Allows easier viewing of motion and manual adjustment
- CSS classes frame-1 through frame-12 reused across all repetitions

Base positioning (frame 1 neutral):
- All CSS positions are base state
- Frame-specific transforms are relative to base positions
- Transform origin on legs: 50% 10% (for knee rotation)

SVG Processing Method (for bitmap exports from Affinity):
1. Export layers from Affinity as SVG (creates embedded base64 PNG)
2. Extract base64 PNG: grep -o 'data:image/png;base64,[^"]*' file.svg
3. Decode to PNG: base64 -D -i base64.txt -o file.png
4. Convert to PNM: magick file.png file.pnm
5. Extract alpha and trace: magick file.png -alpha extract -negate file.pnm
6. Trace with potrace: potrace file.pnm -s -o output.svg
7. Place vectorized files in /public/svg/

Note: Alpha extraction step critical for proper line tracing

Kettlephant Part Sizes (final):
- body.svg: 18KB (was 178KB embedded bitmap)
- eyes.svg: 5.3KB
- handle.svg: 9KB
- leg.svg: 5.9KB
- mouth.svg: 6.3KB
- note.svg: 2KB

JavaScript:
- Scroll position maps to frame index
- Only one frame visible at a time (opacity toggle)
- Smooth scroll-to-frame relationship
